## **1. Concept**

### What is a String?

A **string** is a **sequence of characters** stored together.
Example: `"hello"`, `"abc123"`, `"Tanish"` etc.

In memory, a string is just an **array of characters** ending with a special null character `'\0'` in languages like C/C++.

So:

```cpp
string s = "hello";
```

is basically stored as →
`['h', 'e', 'l', 'l', 'o', '\0']`

---

### **Hinglish Explanation:**

String matlab ek character array jisme letters, digits ya symbols ho sakte hain.
Jaise array mein numbers hote hain, waise string mein characters hote hain.

> “Array of characters = String”

## **2. Why Strings Matter in DSA**

In interviews and problems, strings are used to test:

* Character manipulation (reverse, count, remove)
* Pattern matching
* Palindrome checks
* Frequency or hashing (like anagrams)
* Substrings & subsequences
* Conversions (to int, to char array, etc.)

## **3. Internal Representation**

### In C++:

```cpp
string s = "hello";     // C++ STL string
char arr[] = "hello";   // C-style string (char array)
```

| Aspect          | C-style (`char[]`)       | C++ string (STL)                 |
| --------------- | ------------------------ | -------------------------------- |
| Null terminator | Yes (`'\0'`)             | Hidden (managed internally)      |
| Modifiable size | Fixed                    | Dynamic                          |
| Functions       | `strlen`, `strcpy`, etc. | `s.length()`, `s.substr()`, etc. |

---

## **4. Basic String Operations**

| Operation     | Description                   | Example                       |
| ------------- | ----------------------------- | ----------------------------- |
| Access        | `s[i]` gives i-th character   | `s[0] = 'h'`                  |
| Length        | `s.length()` or `s.size()`    | `"hello".length() = 5`        |
| Substring     | `s.substr(pos, len)`          | `"hello".substr(1,3) → "ell"` |
| Concatenation | `s1 + s2`                     | `"hi" + "bye" = "hibye"`      |
| Comparison    | `s1 == s2`, `s1 < s2`         | `"abc" < "abd"`               |
| Iteration     | `for(char c: s)`              | Print each char               |
| Reverse       | `reverse(s.begin(), s.end())` | `"abc" → "cba"`               |

---

## **5. Memory View (Deep Insight)**

Let’s take:

```cpp
string s = "abc";
```

| Index | 0   | 1   | 2   | 3    |
| ----- | --- | --- | --- | ---- |
| Value | 'a' | 'b' | 'c' | '\0' |

So length is 3, but actual stored characters = 4 (including null).

In modern C++ `string`, you don’t see `'\0'`, but internally it’s there.

---

## **6. Important Notes for Beginners**

1. **Immutable vs Mutable:**

   * In **C++**, `string` is **mutable** (you can change characters).
   * In **Java/Python**, strings are **immutable** (cannot change, must create new string).

2. **Character Encoding:**

   * Each char → stored as ASCII value (integer).
   * `'A'` = 65, `'a'` = 97.
   * Hence `'a' - 'A' = 32` (difference between lowercase & uppercase).

3. **I/O Tricks:**

   * `cin >> s` stops at space.
   * `getline(cin, s)` reads full line with spaces.

## **7. Example: Basic String Operations**

### Problem:

Take a string input and:

1. Print its length
2. Print its reverse
3. Convert to uppercase

### Code:

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    string s;
    cout << "Enter string: ";
    getline(cin, s);  // allows spaces too

    // 1. Length of string
    cout << "Length: " << s.length() << endl;

    // 2. Reverse string
    string rev = s;
    reverse(rev.begin(), rev.end());
    cout << "Reversed: " << rev << endl;

    // 3. Uppercase conversion
    for (char &c : s) {
        c = toupper(c);  // converts each char to uppercase
    }
    cout << "Uppercase: " << s << endl;

    return 0;
}
```

### **Dry Run:**

Input: `"hello world"`

| Step      | Operation       | Output          |
| --------- | --------------- | --------------- |
| Length    | `11`            | `"hello world"` |
| Reverse   | `"dlrow olleh"` |                 |
| Uppercase | `"HELLO WORLD"` |                 |

---

### **Hinglish Explanation:**

* Pehle string liya (with spaces).
* `s.length()` ne character count diya.
* `reverse()` ne last se start tak ulta kar diya.
* `toupper()` ne sab letters ko capital bana diya.

---

## **8. Time Complexity Analysis**

| Operation          | Time Complexity            | Reason |
| ------------------ | -------------------------- | ------ |
| Length (`O(1)`)    | Stored internally          |        |
| Reverse (`O(n)`)   | Every element swapped once |        |
| Uppercase (`O(n)`) | Traverses all characters   |        |

So total = **O(n)** where `n` = string length.
